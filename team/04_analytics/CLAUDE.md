# チーム4：分析AI

## チーム目的
全チームのデータを分析し、具体的な改善提案を行う。
感覚ではなく数字で意思決定できる環境を作る。

## 担当社員
| 社員 | 役割 |
|------|------|
| 社員8（08_x-analytics） | X投稿の反応パターン分析（毎週） |
| 社員9（09_note-analytics） | noteのSEO順位・PV追跡（毎週） |
| 社員10（10_report） | LP改善ポイント分析・週次レポート統括（毎月+毎週月曜） |

## 分析タスク詳細

### 社員8：X分析（毎週）
分析項目：
- 過去7日間の投稿一覧とエンゲージメント（いいね・RT・リプ・インプレッション）
- いいね率・RT率が高かった投稿TOP3の共通パターン抽出
- 反応が悪かった投稿の原因分析
- バズ狙い投稿の成果確認（500いいね達成率）
- 問いかけ投稿のリプ誘発率
- LP誘導投稿のクリック誘発パターン

アウトプット形式：
```
【X分析】週次レポート yyyy/mm/dd〜mm/dd
■ 総投稿数：〇本
■ 平均いいね数：〇
■ バズ投稿達成：〇本（目標：1本以上/週）
■ 反応良かったTOP3：
  1. 「〇〇」→ いいね〇 RT〇
  2.
  3.
■ 共通パターン：〇〇（数字入り / 北海道ネタ / 問いかけ etc）
■ 来週の推奨改善：
  1.
  2.
```

### 社員9：note分析（毎週）
分析項目：
- 記事ごとのPV数・いいね数・コメント数
- 北海道特化キーワードでのSEO順位変動
- 無料→有料記事への流入率
- シリーズ記事の継続読者率
- LP誘導クリック率（記事末尾の誘導効果）

アウトプット形式：
```
【note分析】週次レポート yyyy/mm/dd
■ 今週の記事PV合計：〇PV
■ 上位記事TOP3：
  1. 「〇〇」→ 〇PV
  2.
  3.
■ SEOキーワード順位：
  - 「北海道 副業」：〇位（先週比 ±〇）
  - 「北海道 転職」：〇位（先週比 ±〇）
■ 有料記事クリック率：〇%
■ LP誘導クリック率：〇%
■ 来週の推奨改善：
```

### 社員10：LP分析・レポート統括（毎月+週次統括）
月次LP分析項目：
- LP全体の改善ポイント（文章・構成・CTAの弱点）
- ファーストビューの訴求力評価
- ボタンテキストの改善余地
- 離脱ポイントの推測と対策案
- 成約率向上のための優先改善3項目

週次統括：社員8・9のレポートを統合し、全社への示唆をまとめる

## Slack報告ルール
- 報告チャンネル：#team4-analytics（分析詳細）、#general（週次レポート）
- 週次レポート送信：毎週月曜日 8:00（#generalに送信）
- 月次LP分析：毎月1日（#team4-analyticsに送信、チーム3にも共有）
- 緊急報告：バズ投稿達成時・SEO大幅変動時は即日報告
